# 작업 세분화 (Task Breakdown)

다음 문제를 분석하고 실행 가능한 작은 작업들로 세분화해주세요:

**문제/이슈:**
$ARGUMENT

---

## 분석 및 세분화 지침

다음 구조로 체계적으로 분석해주세요:

### 1. 전체 목표 분석
- 핵심 목표와 최종 결과물이 무엇인지 명확히 정의
- 주요 요구사항 3-5가지 나열
- 제약 조건 및 고려사항 파악
- 현재 프로젝트 컨텍스트와의 연관성 확인

### 2. 작업 세분화
문제를 **3-7개의 주요 작업**으로 나눠주세요. 각 작업은 독립적으로 완료 가능하고 1-2시간 내 완료할 수 있는 크기여야 합니다.

각 작업마다 다음을 포함:

#### 작업 N: [작업명]
- **목적**: 이 작업이 해결하는 구체적 문제
- **작업 내용**:
  - 구체적으로 수행할 작업 나열
  - 생성/수정할 파일 또는 컴포넌트
- **입력**: 이 작업을 시작하기 위해 필요한 것
- **출력**: 이 작업의 완료 시 생성되는 결과물
- **의존성**: 선행되어야 하는 작업 (없으면 "없음")
- **완료 조건**: 이 작업이 완료되었다고 판단하는 기준
- **예상 난이도**: 낮음/중간/높음

### 3. 실행 순서
- 의존성을 고려한 작업 실행 순서 제시
- 병렬로 수행 가능한 작업 그룹 표시
- 각 단계별 검증 포인트 명시

### 4. 잠재적 위험 요소
- 예상되는 기술적 난관
- 불확실한 요구사항 (추가 조사 필요)
- 외부 의존성 (라이브러리, API 등)
- 각 위험에 대한 대응 방안

### 5. 다음 액션
- 지금 바로 시작할 수 있는 첫 번째 작업
- 추가로 명확히 해야 할 질문들 (있다면)
- 사전 준비 사항

---

**참고**: 분석 후 TodoWrite 도구를 사용해 작업 목록을 생성하면 진행 상황을 추적할 수 있습니다.
