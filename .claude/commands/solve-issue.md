# GitHub 이슈 해결 계획 수립 (Solve GitHub Issue)

GitHub 이슈 #$ARGUMENT를 분석하고 해결하기 위한 상세한 실행 계획을 수립합니다.

**이슈 번호:** #$ARGUMENT

---

## 실행 프로세스

다음 단계를 **반드시 순서대로** 수행해주세요:

### 1단계: GitHub 이슈 정보 가져오기

먼저 `gh` CLI를 사용하여 이슈의 전체 정보를 가져옵니다:

```bash
gh issue view $ARGUMENT --json number,title,body,labels,state,author,createdAt,updatedAt,comments
```

**중요**: 이슈 정보를 가져온 후 다음 단계로 진행하세요.

---

### 2단계: 이슈 내용 분석

가져온 이슈 정보를 바탕으로 다음을 분석합니다:

#### A. 이슈 기본 정보
- **이슈 번호**: #$ARGUMENT
- **제목**: [이슈 제목]
- **작성자**: [작성자]
- **생성일**: [날짜]
- **상태**: [Open/Closed]
- **레이블**: [레이블 목록]

#### B. 문제 핵심 파악
- 이슈가 다루는 **핵심 문제**가 무엇인가?
- 이슈 유형: 버그 수정 / 새 기능 / 개선 / 리팩토링 / 문서화
- 영향 범위: 어떤 파일/컴포넌트/모듈이 관련되는가?
- 우선순위: 긴급 / 높음 / 중간 / 낮음

#### C. 요구사항 추출
1. 필수 요구사항 (Must Have)
2. 선택적 요구사항 (Nice to Have)
3. 제약 조건 및 고려사항
4. 성공 기준 (Definition of Done)

---

### 3단계: 현재 코드베이스 조사

이슈 해결에 필요한 파일과 코드를 조사합니다:

1. **관련 파일 검색**
   - Glob, Grep 도구를 사용하여 관련 파일 찾기
   - 주요 컴포넌트/모듈 위치 파악

2. **기존 구현 파악**
   - 관련된 기존 코드 읽기
   - 현재 구조와 패턴 이해
   - 의존성 파악

3. **유사 사례 검색**
   - 비슷한 기능이 이미 구현되어 있는지 확인
   - 재사용 가능한 컴포넌트/함수 탐색

**중요**: Task 도구를 사용하여 코드베이스를 조사할 수 있습니다.

---

### 4단계: 해결 계획 수립

다음 구조로 **상세한 실행 계획**을 작성합니다:

## 🎯 이슈 해결 계획

### 📋 이슈 요약
[이슈의 핵심 내용을 2-3문장으로 요약]

- **현재 상황**: [현재 어떻게 동작하는지 또는 무엇이 부족한지]
- **목표 상황**: [이슈 해결 후 어떻게 되어야 하는지]
- **주요 과제**: [해결해야 할 핵심 문제들]

- **선택한 방법**: [어떤 방식으로 구현할 것인지]

1. **1단계**: 작업 1, 작업 2 (병렬 가능)

3. **3단계**: 작업 4, 작업 5 (병렬 가능)

### 5단계: 사용자 승인 요청

계획을 모두 작성한 후, **반드시** 다음을 수행합니다:

1. **AskUserQuestion 도구 사용** - 사용자에게 승인을 요청합니다:

```
질문: "위 해결 계획을 검토해주세요. 진행해도 괜찮을까요?"
옵션:
- "승인 - 계획대로 진행하세요"
- "수정 필요 - 피드백을 주겠습니다"
- "다른 방법 제안 - 대안을 제시하겠습니다"
```

2. **사용자 응답에 따라 행동**:
   - **승인**: TodoWrite 도구로 작업 목록을 생성하고 구현 시작
   - **수정 필요**: 사용자 피드백을 반영하여 계획 수정
   - **다른 방법**: 사용자의 제안을 듣고 계획 재수립

---

## 📌 중요 지침

### 반드시 지켜야 할 사항:
1. ✅ 1단계에서 **반드시** `gh issue view` 명령어로 이슈 정보를 먼저 가져올 것
2. ✅ 코드베이스 조사를 **실제로** 수행할 것 (추측하지 말 것)
3. ✅ 계획은 **구체적이고 실행 가능**하게 작성할 것
4. ✅ 5단계에서 **반드시** AskUserQuestion으로 사용자 승인을 받을 것
5. ✅ 승인받기 전에는 **절대** 코드 수정을 시작하지 말 것

### 계획 작성 팁:
- 각 작업은 1-2시간 내 완료 가능한 크기로 분할
- 파일 경로는 정확하게 명시
- "아마도", "대충" 같은 모호한 표현 지양
- 구체적인 함수명, 컴포넌트명 사용
- 의존성을 명확히 표시

### 사용자 승인 후:
- TodoWrite 도구로 작업 목록 생성
- 첫 번째 작업부터 순차적으로 진행
- 각 작업 완료 시 체크리스트 업데이트

---

## 예시 워크플로우

**입력**: `/solve-issue 42`

**실행**:
1. `gh issue view 42 --json ...` 실행 → 이슈 정보 확인
2. 이슈가 "로그인 페이지에 소셜 로그인 추가" 요청임을 파악
3. 관련 파일 검색 (`app/login/page.tsx`, `providers/auth-provider.tsx` 등)
4. 상세 계획 수립 (OAuth 설정, UI 추가, 인증 로직 구현 등)
5. 사용자에게 계획 승인 요청
6. 승인 받으면 → TodoWrite로 작업 목록 생성 → 구현 시작
