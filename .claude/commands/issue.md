# GitHub 이슈 생성 (Create GitHub Issue)

다음 내용을 바탕으로 체계적인 GitHub 이슈를 생성해주세요:

**이슈 내용:**
$ARGUMENT

---

## 이슈 생성 지침

다음 단계를 수행해주세요:

### 1. 이슈 내용 분석
- 핵심 문제 또는 기능 요청이 무엇인지 파악
- 이슈 유형 결정 (버그, 기능 요청, 개선, 문서화, 질문 등)
- 우선순위 평가 (높음, 중간, 낮음)

### 2. 이슈 제목 작성
- 명확하고 간결한 제목 (50자 이내 권장)
- 이슈 유형을 반영한 접두사 사용 (선택사항):
  - `[Bug]`: 버그 수정
  - `[Feature]`: 새 기능
  - `[Enhancement]`: 기능 개선
  - `[Docs]`: 문서화
  - `[Question]`: 질문

### 3. 이슈 본문 작성
다음 구조로 작성해주세요:

#### 버그 리포트의 경우:
```markdown
## 문제 설명
[간단한 문제 설명]

## 재현 방법
1. [첫 번째 단계]
2. [두 번째 단계]
3. ...

## 예상 동작
[어떻게 동작해야 하는지]

## 실제 동작
[현재 어떻게 동작하는지]

## 환경 정보
- OS:
- 브라우저/Node 버전:
- 기타:

## 추가 정보
[스크린샷, 에러 로그 등]
```

#### 기능 요청의 경우:
```markdown
## 기능 설명
[구현하고자 하는 기능]

## 동기 및 배경
[왜 이 기능이 필요한지]

## 제안 방안
[어떻게 구현할 수 있을지]

## 대안
[고려한 다른 방법들]

## 추가 컨텍스트
[관련 정보, 참고 자료 등]
```

#### 개선/기타의 경우:
```markdown
## 개선 내용
[무엇을 개선할 것인지]

## 현재 상황
[현재 어떻게 되어 있는지]

## 개선 후 기대 효과
[개선하면 어떤 장점이 있는지]

## 구현 방안
[어떻게 구현할 것인지]
```

### 4. 레이블 제안
적절한 레이블을 제안해주세요:
- `bug`: 버그 관련
- `enhancement`: 기능 개선
- `feature`: 새 기능
- `documentation`: 문서화
- `question`: 질문
- `good first issue`: 초보자에게 좋은 이슈
- `help wanted`: 도움이 필요함
- `priority-high`: 높은 우선순위
- `priority-medium`: 중간 우선순위
- `priority-low`: 낮은 우선순위

### 5. GitHub 이슈 생성
`gh` CLI를 사용하여 이슈를 생성해주세요:

```bash
gh issue create \
  --title "[이슈 제목]" \
  --body "$(cat <<'EOF'
[이슈 본문 전체 내용]
EOF
)"
```

필요한 경우 레이블도 함께 추가:
```bash
gh issue create \
  --title "[이슈 제목]" \
  --body "..." \
  --label "bug,priority-high"
```

### 6. 완료 후 확인
- 생성된 이슈 URL 반환
- 이슈 번호 확인
- 사용자에게 이슈 링크 제공

---

## 주의사항
- 이슈 제목은 명확하고 구체적으로
- 본문은 충분한 컨텍스트를 포함
- 재현 가능한 정보 제공 (버그의 경우)
- 마크다운 형식 사용
- 코드 블록에는 적절한 언어 지정

## 예시

**입력:** "로그인 페이지에서 비밀번호를 잘못 입력해도 에러 메시지가 안 나와요"

**출력:**
- 제목: `[Bug] 로그인 실패 시 에러 메시지 미표시`
- 본문: 문제 설명, 재현 방법, 예상/실제 동작 포함
- 레이블: `bug`, `priority-high`
- `gh issue create` 명령어 실행
